Windows下管理文件的方法

发表于 2015-09-13 | 分类于 软件 | |

写一下我在Windows下管理文件的方法

本文写给, 用Windows但没有形成自己的一套管理文件的逻辑的人. 特别是需要管理相当大数量的文件的人, 比如我的非系统和应用程序类文件全部在D盘, 现在有22万个左右, 这些文件的特征是: 基本都是有用的, 没什么垃圾文件, 因为我有定期清理不需要文件的习惯; 都是琐碎的小文件, 我的硬盘里几乎没有视频 (这是最好管的文件了, 我把它们放在大容量移动硬盘), D盘用了200+GB, 这意味着我的文件平均在几MB大小, 而大量的小文件是最难管理的. 所以我觉得我的管理方法基本适用于所有情况.

我的两个分区, D盘全部目录, 最重要的两个文件夹:桌面和我的文档

从大往小说.

下次再补充一些工具介绍, 加强资源管理的比如listary, clover, everything, TC等. 另外资源管理器的能力有限, 如果是比较单一的文件种类最好用相应库管理软件, 比如音乐用iTunes, 图书用Calibre, 代码用Git.

##分区##

我觉得现在没有把本地磁盘分区分到2个以上的必要, 把系统和所有程序装在C盘, 剩下都放在D盘, 相对于多分区的好处是显然的, 随便说两个:

  • 在同一个分区之内剪切/移动文件是瞬间完成的, 原理想一下就知道, 它只需要改几个位置的某些字符串. 但在分区之间剪切或复制内容的时候, 那就是实实在在地创建新文件.
  • 多分区的用户往往思维混乱, 比如他们把D盘起个名字叫工作, E盘起个名字叫娱乐, 发现E盘放不下了, 就搞个F盘起个名字叫电影, 各分区之间经常有很多功能重复的目录, 结果就是一个文件放这里也可以, 放哪里也可以, 那么要找文件的时候, 也就是从这里找也可以, 从那里找也可以, [[这是任何工具都帮不了你的问题,everything,]] listary什么的都帮不了. 将系统和应用程序以外文件全部放在一个盘, 才能随心所欲地管理文件.

综上所述, 把磁盘分为两个

一个装系统, 比如C盘, 大多数情况下, 不应该直接操作C盘, C盘有改动对于一般用户来说只能是以下几个操作: 安装和卸载程序, 重装系统.
一个放其他东西, 比如D盘, 因为C盘不需要我们操心, 所以本文以下基本都是在说D盘的文件管理

##D盘:##

###Cache###

这是很重要的文件夹, 所有浏览器和下载工具的下载目录全部设定到这个文件夹, 同时把这个文件夹设定为桌面文件夹 (即桌面上显示的就是这个文件夹的内容, 看图就清楚了), 平时下载的文件, 写代码时需要用来测试的文件, 还有其他临时文件全部在这个文件夹, 现在它只有一个文件因为我整理过了, 平时是可以很乱的, 它提醒你赶紧把这些玩意处理掉, 需要保留的就把它剪切到相应位置, 需要删的就shift+delete.

###WindowsImageBackup###

这是Win7自带还原生成的, 这个看个人习惯, 我之前用Comodo Time Machine, 尽管很快, 但是会造成一些问题, 所以我还是用回win自带的, 注意把备份周期设置地长一点, 否则会占用大量空间

###Portable&Bin###

翻译成中文就是绿色软件, 绿色软件(Portable Software)的初衷是Portable, 装在U盘里到处都能用(因此我用Dropbox同步和备份这个文件夹), 但我看中的是:

  • Portable Software 往往是开源的, 免费的, 安全的, 小巧的
  • 它可以在你无数次重装系统后, 无需安装即可打开, 并且和linux一样保留了用户习惯, 无需重新配置, 还是熟悉的配方.
    所以建议能用Portable的都用.

一般来说很多软件自己会有Portable Ver供下载, 也会有一些人专门把软件绿色化, 但一定要找安全的来源, 以下几个是常用的找绿色软件的网站

  • 精品绿色便携软件
  • 小众软件
  • PortableApps

Bin目录则是linux下的一些工具

####子文件夹####

单纯的列表, 每个软件一个文件夹

####同步和备份####

网络备份: Dropbox
本地备份: 备份到TrueCrypt加密过的U盘中

###Develop###

原来我是把自己的代码放在Documents\Develop下的, 但是后来我把它单独拿出来了, 原因是, 我的Document文件夹是用Dropbox同步的, 常年开着Dropbox, 每改几行一保存一编译就同步一次各种代码中间文件编译结果等实在太蛋疼了, 尽管我也可以用某些方法让它不同步这个目录, 但还是拿出来方便一些. 而且其他文档我需要Dropbox的版本控制, 但那是针对一般文档, 比起git等专用于代码的版本控制系统来说, Dropbox就显得弱爆了, 我不愿意Dropbox把我的git目录也一起同步上去, 所以我的代码都是push到github或者bitbucket作为备份.

####子文件夹####

具体分类是这样的, 如果是针对平台的开发, 就放在对应平台文件夹下, 比如Windows/ARM等, 如果没什么特殊平台, 用同一种语言开发, 就放在对应语言文件夹下, 比如AHK/JAVA等

每个文件夹下如图所示

分Documents和Projects文件夹, 一些还有Library/SDK文件夹, Documents就是对应文档, 比如j2se 的API文档等, Projects就是一个个项目目录了, 都用git管理.

####同步和备份####

  • 网络备份: 如上所述, 私有的push到bitbucket, 共有的到github
  • 本地备份: 备份到TrueCrypt加密过的U盘中
  • ##库##

Win7的库是很棒的设计, 它本身没什么值得一提的功能, 但是告诉用户一个整理自己文件的基本分类法, 即使你把文件像我上面说的一样乱放, 它也能在一定程度上帮助你弄清楚自己的文件结构. 但这个功能没什么人用…

最好在把库的位置修改到D盘下, 这样就把全部的文件操作局限在D盘, 移动起来很方便.

这是我的D盘, 可以看出它是按照库的形式来管理的, Audio对应我的音乐库, Document对应我的文档.

###Image###

####子文件夹####

  • Lightroom文件夹是Lightroom的一些配置和数据库, 放在这里重装系统以后载入即可.
  • Photo下是照片, 按年份分文件夹, 但实际上不怎么用资源管理器管, 都是开Lightroom管理文件夹
  • Picture是非照片类的图片, 比如各种从网上拖的图片, 其他人拍的照片, 截图等等
  • PS下就是PS用的素材和PS项目
  • ####同步和备份####
  • 网络备份: 由于文件较大, 用百度云备份和同步
  • 本地备份: 同时备份到移动硬盘中
    ###Audio###

我一开始是想直接在Audio下放音乐的, 但是还有一些其他文件不应该被混淆到音乐中, 所以这样分一下

####子文件夹####

  • iTunes和Xmusic是iTunes虾米音乐的配置目录, 不管它
  • Ringtones是自己做的一些铃声
  • CD是整轨的无损音乐, 全部手工整理好cue和封面放在里面
  • music就是音乐文件了, 还是一样因为数量比较大, 在几千首歌里找自己想要的歌用文件管理器是找死, 用iTunes管理它, 下载好歌曲到Cache文件夹后配好内置歌词和封面, 修改好ID3后再放到这个文件夹里.
    ####同步和备份####
  • 网络备份: 还是文件较大, 用百度云
  • 本地备份: 备份到移动硬盘中
    ###Vedio###

基本上除了自己剪辑的几个小视频和几个演讲以外什么都没放, 把视频放到笔记本的本地磁盘越来越不可能了, 只能外接移动硬盘或NAS放在1TB\2TB的移动硬盘中, 我现在电影和动漫以及其他视频加起来1TB, 马上又得买新盘….

###Document###

当时想用单数, 现在想改回主流用法Documents, 涉及改动比较大, 就算了.

####子文件夹####

这比较个人化, 没什么好说的, 我挑几个说

  • Book 这里面大概有几十书, 是我已经读过或者将要读的书, 我的图书馆Library在D:\Keep文件夹下, 里面有几千本书, 用Calibre管. 不过我一般不在PC上看书, 这个单纯是为了放在

  • Dropbox下可以在各平台看

  • System 放一些跟系统有关的备份, 比如字体, 主题, 某些软件的配置文件, 皮肤等
    还有My knowledge这些是软件自己生成的目录.

####同步和备份####

  • 网络备份: Dropbox
  • 本地备份: 加密的U盘中
