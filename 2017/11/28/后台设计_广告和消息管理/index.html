一、项目概述

1.1项目背景

目前老管理后广告管理和需求管理功能较弱，同时也需要将功能模块迁移至新运营管理后台，借此机会优化广告管理和需求管理模块并迁移，同时满足运营、客服部门对广告和消息操作配置需求。

1.2词汇表

序号** 词汇** 解释**
1 广告管理 通过新运营管理后台管理和配置317护管理后台、护士 App 端、微信端资源位。
2 消息管理 通过新运营管理后台管理和配置 App 内推送、患者消息、微信消息、短信通道。

2.2通用模块

2.2.1 发送对象

模块概述 发送对象设置模块，广告管理和消息管理通用。
界面原型
规则详情 1、发送对象支持按全部、按指定医院、自定义条件设置，默认发送对象为全部： 全部：选中所有医院； 按指定医院：支持按关键词搜索医院，支持全选、多选（反选），与目前老运营后台操作保持一致即可； 自定义条件：支持按照医院地区、等级、类型、对象进行设置，均默认全选，支持多选。 • 地区：省份、地级市 • 等级：三级特等、三级甲等、三级乙等、三级丙等、二级甲等、二级乙等、二级丙等、一级甲等、一级乙等、一级丙等 • 类型：儿童医院、妇产医院、综合医院、中医院、慢病医院、其它专科医院 • 对象：医院管理员、病区管理员、科室管理员、护士。 注意： *该模块为通用模块

2.2.2 内容设置

模块概述 内容设置
界面原型
规则详情 1、内容设置支持跳转至URL、自定义内容、无内容设置，默认选中跳转至URL； 2、跳转至URL需要配置链接，选中但为空，弹窗提醒”请输入URL地址”； 3、自定义内容支持HTML编辑器自主输入内容，可视化操作； 4、无内容则点击没有反应。 注意：*该模块为通用模块

2.2.3 消息推送时间弹窗

模块概述 消息推送时间设置弹窗
界面原型
规则详情 1、点击推送是弹窗显示该页面； 2、推送时间有立即推送和定时推送可选，立即推送为立即执行操作； 4、定时推送按照预约时间进行推送，时间精确到到秒，形如 2017-11-08 11:21:38 注意：*该模块为通用模块

2.3.2 消息管理

2.3.2.1 App 内推送

功能概述 新运营后台后台配置 护士App 内消息推送
界面原型
前置流程 打开【运营后台】-【管理医院】-【消息管理】可见
功能说明 展示消息列表
规则详情 列表规则： 1、记录消息 标题、发送时间、到达/发送任务、发送人、状态 的信息； 2、展示列表以发送时间先后从上到下排序（刚发送的展示在最前面）； 3、消息状态包括 正在推送 、已完成、 待推送、被停止 四种： 正在推送——消息已经执行推送，正在推送中 已完成——消息已经全部完成推送 待推送——消息已经编辑设置完成，保存完毕，等待执行推送操作 被停止——消息推送中强行停止 4、对单条消息可进行 查看、停止、推送、编辑、删除6种操作： 查看—查看处于正在推送和已完成状态的消息详细内容，不可进行编辑和删除操作 停止——停止处于正在推送状态的消息 推送——对处于待推送状态的消息执行推送操作 编辑——编辑处于待推送和被停止状态的消息 删除——删除处于待推送和被停止状态的消息 5、点击【推送】显示推送时间设置弹窗 推送时间弹窗为通用模块 新增编辑规则： 1、标题：必填项，最多15个字； 2、封面上传：图片建议尺寸为 900500 px； 3、内容设置：通用模块，保持一致； 4、推送范围设置：通用模块，但新增按制定账号推送，规则：纯文本框；换行区分不同手机号。 * 新增按制定账号推送
